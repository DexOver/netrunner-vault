# NETRUNNER VAULT

Локальный менеджер паролей. Полностью оффлайн. Никаких облаков.

---

## ЧТО ИСПРАВЛЕНО — БЕЗОПАСНОСТЬ

### Критичное

1. **Удалено хранение мастер-пароля в `sessionStorage`.** Это была главная дыра — пароль лежал открытым текстом в памяти браузера и был доступен любому XSS. Теперь пароль вводится при каждом входе и нигде не сохраняется.
2. **Автоблокировка при сворачивании приложения** (`visibilitychange`). Когда сворачиваешь — приложение лочится мгновенно.
3. **Автоблокировка по таймауту бездействия** — 5 минут.

### Криптография

4. **PBKDF2 итераций: 250 000 → 600 000** (минимум OWASP 2023+ для SHA-256).
5. **Длина соли: 16 → 32 байта** (256 бит).
6. **Параметры KDF теперь хранятся в самом vault** (`v`, `iter`, `kdf`). Это даёт:
   - совместимость со старыми бэкапами,
   - возможность безопасно повышать стоимость в будущем.
7. **Автомиграция старых vault** при первом успешном входе на новые параметры.
8. **Минимальная длина мастер-пароля: 6 → 10 символов** + проверка надёжности (нужны минимум 3 категории символов).
9. **Защита от brute-force.** После 3+ неудачных попыток включается экспоненциальная задержка: 2с → 10с → 30с → 120с.
10. **Исправлен `genPw()`.** Старая версия использовала `n % 72` на 32-битном случайном числе — это давало неравномерное распределение (modulo bias). Теперь rejection sampling.
11. **Длина генерируемого пароля: 18 → 20 символов** по умолчанию.
12. **Исправлен `b64e()`.** Старая версия (`String.fromCharCode(...u)`) падала со stack overflow на больших vault. Теперь chunk-based.

### Сеть и изоляция

13. **CSP (Content Security Policy)** в `<head>`: блокирует все внешние запросы, inline-eval запрещён, frame-ancestors none.
14. **`referrer: no-referrer`** — никаких утечек через Referer.
15. **Удалён Google Fonts** — приложение действительно работает оффлайн, как и обещает splash. Используется системный шрифтовый стек.
16. **Service Worker блокирует любые внешние запросы** на уровне fetch handler.

### Прочее

17. **Автоочистка clipboard через 30 секунд** после копирования пароля. Перед очисткой проверяется, что в буфере всё ещё наш пароль (если ты что-то скопировал после — не трогаем).
18. **Экспорт через Blob** вместо `data:` URL — работает с большими vault, не попадает в историю браузера.
19. **Поддержка PWA**: manifest, иконки, service worker. Можно установить как приложение.

## ЧТО ОСТАЛОСЬ ОГРАНИЧЕНИЕМ ПЛАТФОРМЫ

Эти вещи **в принципе нельзя** сделать в вебе/PWA:

- **JavaScript-строки иммутабельны.** Расшифрованные пароли остаются в памяти процесса до GC. В нативном приложении можно было бы использовать `mlock`/`SecureString`. На вебе — нет.
- **Дамп памяти процесса.** Если у атакующего root на устройстве — он сможет дампнуть память Chrome и достать ключ из `cKey`. Это касается всех веб-парольников без исключения (включая Bitwarden web, 1Password web).
- **Скриншоты экрана.** В нативном Android-приложении можно поставить `FLAG_SECURE`. В PWA — нельзя. После сборки в APK через PWABuilder это можно добавить вручную в Android Studio.
- **Биометрия (отпечаток).** WebAuthn в PWA имеет ограниченную поддержку и не даёт надёжного дополнительного фактора для шифрования. Лучше пока оставить только мастер-пароль.

## КАК ПОЛЬЗОВАТЬСЯ ЛОКАЛЬНО (без APK)

1. Распакуй все файлы в одну папку.
2. Открой `index.html` в браузере.

Для PWA-функций (оффлайн через service worker) нужен HTTPS либо `localhost`. Для теста:

```bash
cd netrunner
python3 -m http.server 8080
# открой http://localhost:8080
```

## КАК СОБРАТЬ APK ЧЕРЕЗ PWABUILDER

PWABuilder требует, чтобы приложение было доступно по HTTPS. Простейший путь:

### Вариант А: GitHub Pages (бесплатно, рекомендую)

1. Создай новый репозиторий на GitHub, например `netrunner-vault`.
2. Загрузи туда все файлы из этой папки (`index.html`, `manifest.webmanifest`, `sw.js`, иконки).
3. В настройках репозитория → **Pages** → выбери ветку `main`, папка `/` (root). Сохрани.
4. Через 1–2 минуты получишь URL вида `https://твой-логин.github.io/netrunner-vault/`.
5. Открой [https://www.pwabuilder.com](https://www.pwabuilder.com).
6. Вставь туда URL, нажми **Start**.
7. PWABuilder проверит manifest и SW — должно быть 100% (всё уже подготовлено).
8. Нажми **Package For Stores → Android**.
9. Скачай ZIP. Внутри будет `app-release-signed.apk`.

### Вариант Б: локальный HTTPS через `ngrok`

Если не хочешь публиковать:

```bash
cd netrunner
python3 -m http.server 8080
# в другом терминале:
ngrok http 8080
```

Используй HTTPS-URL от ngrok для PWABuilder. Минус: URL временный.

### После сборки APK

1. На Android разреши установку из неизвестных источников.
2. Скопируй APK на телефон, открой, установи.
3. Иконка появится в меню. **Работает полностью оффлайн** — service worker кэширует всё на первой загрузке.

### Дополнительно (если откроешь APK в Android Studio)

В `app/src/main/AndroidManifest.xml` или в Activity:

```kotlin
window.addFlags(WindowManager.LayoutParams.FLAG_SECURE)
```

Это запретит скриншоты и появление окна в превью задач — рекомендую для парольника.
